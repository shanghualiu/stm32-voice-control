# 基于离线语音识别的家电控制系统 — 使用说明与提交指南

## 📋 一、快速开始

### 🚀 方式一：GitHub Actions 在线编译（✅ 推荐，零安装）

**无需安装任何软件！** 只需将代码推到 GitHub，自动编译出 hex 文件。

#### 操作步骤

1. **创建 GitHub 仓库**：在 github.com 新建一个仓库（可设为 Private）

2. **推送代码**：
   ```bash
   cd Voice_Control_System
   git init
   git add .
   git commit -m "Initial commit: Voice Control System"
   git remote add origin https://github.com/你的用户名/你的仓库名.git
   git push -u origin main
   ```

3. **等待自动编译**（约 2~3 分钟）：
   - 进入仓库页面 → 点击 **Actions** 标签页
   - 看到 "Build STM32 Firmware" 工作流正在运行
   - 运行完成后变成绿色 ✅

4. **下载 hex 固件**：
   - 点击完成的工作流 → 页面底部 **Artifacts** 区域
   - 点击 **`Voice_Control_System_Firmware`** 下载 zip 包
   - 解压得到 `Voice_Control_System.hex`

5. **导入 Wokwi 仿真**：
   - 打开 https://wokwi.com → 新建 STM32 Blue Pill 项目
   - 将 `wokwi/diagram.json` 内容复制到 diagram.json 标签页
   - 将下载的 `.hex` 文件拖入 Wokwi 文件列表
   - 或在 diagram.json 中设置 `"firmware": "Voice_Control_System.hex"`
   - 点击 ▶ **Start Simulation**

> 💡 **手动触发编译**：在 Actions 页面点击 "Build STM32 Firmware" → "Run workflow" → "Run workflow" 绿色按钮即可随时重新编译。

### 💻 方式二：本地 ARM GCC 编译（免费，需安装工具链）

```bash
# 1. 安装 ARM GCC 工具链
#    Ubuntu/Debian: sudo apt-get install gcc-arm-none-eabi
#    macOS:         brew install arm-none-eabi-gcc
#    Windows:       下载 GNU Arm Embedded Toolchain

# 2. 下载 HAL/CMSIS 依赖（仅需执行一次）
bash fetch_hal_deps.sh
# 或: make setup

# 3. 编译
make clean && make

# 4. 输出文件在 build/ 目录下
#    Voice_Control_System.hex ← Wokwi 用
```

### 🖥️ 方式三：Keil MDK 本地编译（需安装 Keil 5.38a+）

#### 前提条件
- Keil MDK 5.38a+
- STM32F1 Device Family Pack（通过 Pack Installer 安装）
- STM32CubeF1 HAL 库

#### 工程创建步骤

1. **创建新工程**：Project → New μVision Project → 芯片: STM32F103C8

2. **添加源文件组**：
   - 添加 `Core/Src/` 下所有 .c 文件
   - 添加 `Core/Inc/` 下所有 .h 文件
   - 添加 HAL 库 src 文件

3. **配置 Include Paths**：
   ```
   Core/Inc
   Drivers/CMSIS/Include
   Drivers/CMSIS/Device/ST/STM32F1xx/Include
   Drivers/STM32F1xx_HAL_Driver/Inc
   ```

4. **配置宏定义**：`STM32F103xB,USE_HAL_DRIVER`

5. **编译**：F7 → 输出 `MDK-ARM/Debug/Voice_Control_System.hex`

在 Wokwi Serial Monitor 输入框选择 **Hex** 模式，输入：

| 指令 ID | 语音命令 | 效果 |
|---------|----------|------|
| `01` | 打开灯光 | CH1 继电器吸合, 黄 LED 亮 |
| `02` | 关闭灯光 | CH1 继电器断开, 黄 LED 灭 |
| `03` | 打开风扇 | CH2 继电器吸合, 绿 LED 亮 |
| `04` | 关闭风扇 | CH2 继电器断开, 绿 LED 灭 |
| `05` | 打开插座 | CH3 继电器吸合, 蓝 LED 亮 |
| `06` | 关闭插座 | CH3 继电器断开, 蓝 LED 灭 |
| `07` | 打开备用 | CH4 继电器吸合, 红 LED 亮 |
| `08` | 关闭备用 | CH4 继电器断开, 红 LED 灭 |
| `09` | 全部打开 | CH1~4 全部吸合, 全部 LED 亮 |
| `0A` | 全部关闭 | CH1~4 全部断开, 全部 LED 灭 |

## ❓ 六、常见问题 FAQ

### Q1: 编译时报 "stm32f1xx.h not found"？
**A**: 检查 Keil → Options → C/C++ → Include Paths 是否正确添加了 CMSIS 和 HAL 库路径。

### Q2: Wokwi 中 OLED 不显示？
**A**: 检查：
1. `diagram.json` 中 SCL/SDA 连接是否正确（PB6/PB7）
2. I2C 地址是否正确（0x3C，即 0x78）
3. 是否已上传 hex 固件

### Q3: 串口发送 01 无反应？
**A**: 确认 Serial Monitor 设置为 **Hex 模式**（不是 Text 模式），然后输入 `01`（不带 0x 前缀）。

### Q4: 按键按下 LED 不变化？
**A**: 按键消抖需要 20ms，快速点击可能被消抖过滤。正常速度点击（按下 > 30ms）即可触发。

### Q5: 如何从仿真迁移到实物？
**A**: 仅需修改两处：
1. 添加 LD3320 的 SPI 驱动代码，替换 USART1 数据源
2. 确认继电器模块的触发电平（低电平/高电平），调整 `Relay_Ctrl()` 中的电平映射
3. OLED、按键等模块代码无需任何修改
